#### CommonsChunkPlugin --- webpack4 之前使用

#### webpack.optimization.splitChunks

webpack4 及其以后的版本使用

```
const path = require("path");
module.exports = {
  entry: "./src/index.js",
  output: {
    path: path.join(__dirname, "dist"),
    filename: "[name].bundle.js"
  },
  devServer: {
    port: 8000,
    publicPath: "/dist/"
  },
  optimization: {
    splitChunks: {
      // chunks: async(默认，只提取异步模块) | initial(只提取入口) | all(前两者都提取)
      // 如果一个文件被引入大于等于两次，那么就会被打包成一个单独的chunk，chunks: "all" 标记对所有的chunk都生效
      chunks: "all", // SplitChunks将会对所有的chunks生效，默认情况下只对异步chunks生效
      // 按cacheGroups的提取规则，并以automaticNameDelimiter为分隔符命名chunks
      // eg: vendors~a~b~c.js意思是该chunk为vendors规则所提取，并且该chunk是由a、b、c三个入口chunk所产生的。
      name: true,
        /*
        * 根据chunk来源配置提取规则
        */
        cacheGroups: {
            // 模块来自node_modules目录，vendors只是chunk命名，可灵活调整；
            vendors: {
                test: /[\\/]node_modules[\\/]/,
                priority: -10, // 优先级，这里vendors优先
            },
            // chunk被至少两个模块引用则重用
            default: {
                minChunks: 2,
                reuseExistingChunk: true,
                priority: -20,
            },
        }
    }
  },
  mode: "development"
};
```

index.js

```

import React from "react";
import "./app";
console.log("index.js React Verion: ", React.version);
```

app.js

```
import React from "react";
console.log("app.js React Verion: ", React.version);
document.write("<h1>hello App.js</h1>");
```

打包结果

```
asset vendors-node_modules_react_index_js.bundle.js 77 KiB [compared for emit] (id hint: vendors)
asset main.bundle.js 13.3 KiB [emitted] (name: main)
asset src_app_js.bundle.js 1.21 KiB [emitted]
```

webpack 打包之后会发现除了生成了 main.bundle.js 还生成了一个 vendors-node...的文件，里面打包了 react

##### splitChunks 更像是声明式的

**一般配置了 chunks: 'all'选项即可**

默认的提取条件

- 提取后的 chunk 可以被共享或者来自 node_modules 目录

- 提取的 js chunk 大小大于 30kb ,css chunk 大小大于 50kb，压缩前

- 按需加载的时候，资源并行请求的个数最大值小于等于 5

- 首次加载时，并行加载的资源数的最大值小于等于 3

#### 资源异步加载

在 webpack 中有两种异步加载资源的方式

1. 动态的 import (import 函数) 推荐使用

2. require.ensure webpack 1 中的

##### import

webpack.config.js

```
const path = require("path");
module.exports = {
  entry: "./src/index.js",
  output: {
    path: path.join(__dirname, "dist"),
    filename: "[name].bundle.js",
    publicPath: "/dist/"
  },
  devServer: {
    port: 8000,
    publicPath: "/dist/"
  },
  mode: "development"
};
```

index.js

```
import("./app").then(() => {
  console.log("load app");
});

```

app.js

```
import React from "react";
console.log("app.js React Verion: ", React.version);
document.write("<h1>hello App.js</h1>");
```

首屏加载 js 的地址是通过 script 标签来决定的，而间接资源（通过首屏 js 再加载进来的资源）的位置则要通过 output.publicPath 来指定 (和 output.path 保持一致)

**import 函数还有一个重要的特点，es6 要求 import 必须出现在顶层作用域，但是 webpack 的 import 可以出现在任意地方，或者是条件语句中**

**原理：通过 js 动态地在页面 head 标签内插入一个 script 标签，并通过 publicPath 拼接其 url**

##### 异步的 chunk 配置

1. 指定异步加载的 chunk 的文件名

webpack.config.js

新增 output.chunkFilename 选项，配置命名规则

```
const path = require("path");
module.exports = {
  entry: "./src/index.js",
  output: {
    path: path.join(__dirname, "dist"),
    filename: "[name].bundle.js",
    publicPath: "/dist/",
    chunkFilename: "[name].js"
  },
  devServer: {
    port: 8000,
    publicPath: "/dist/"
  },
  mode: "development"
};
```

index.js

在动态导入的 import 前加上一句特殊的注释，用来指定 chunkname

/\* webpackChunkName: 'bar' \*/

```
import(/* webpackChunkName: 'bar' */ "./app").then(() => {
  console.log("load app");
});
```

#### 总结

1. 手动配置代码分割：在 webpack4 之后我们使用 optimization.splitChunks.chunks = 'all' 可以让 webpack 为我们将公共模块提取为单独的 chunk，一般设置 chunks: 'all' 就可以达到很好的代码分割标准

2. 异步加载的模块，webpack 会默认帮我们进行代码分割：而且对于异步加载的模块(import('./app.js'))这种动态引入的模块，默认不配置 optimization.splitChunks.chunks webpack 也可以为我们进行代码分割（将 node_modules 下的模块单独进行打包到一个 bundle 中）
