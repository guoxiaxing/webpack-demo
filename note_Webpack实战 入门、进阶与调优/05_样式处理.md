#### 分离样式文件

将 style 标签中的样式抽离到单独的 css 文件当中

mini-css-extract-plugin webpack 4 版本及其以上

extract-text-webpack-plugin webpack 4 之前的版本

样式的提取是以资源入口文件开始的整个 chunk 而言的，所以对于多入口文件我们需要注意提取到的样式文件可能会覆盖的问题

##### mini-css-extract-plugin

支持按需加载 css

举个例子，a.js 通过 import()函数异步加载了 b.js，b.js 里面加载了 style.css，那么 style.css 最终只能被同步加载（通过 HTML 的 link 标签）。但是现在 mini-css-extract-plugin 会单独打包出一个 0.css（假设使用默认配置），这个 CSS 文件将由 a.js 通过动态插入 link 标签的方式加载。

```
const path = require("path");
const MiniExtractWebpackPlugin = require("mini-css-extract-plugin");
module.exports = {
  entry: "./src/app.js",
  output: {
    path: path.join(__dirname, "dist"),
    filename: "bundle.js"
    // publicPath: "./assets/"
  },
  // 与loader相关的配置都在 module.rules 中
  module: {
    rules: [
      {
        test: /\.ts$/,
        use: "ts-loader"
      },
      {
        test: /\.html$/,
        use: "html-loader"
      },
      {
        test: /\.(jpg|png|gif)$/,
        loader: "url-loader",
        options: {
          limit: 10240000,
          name: "[name].[ext]",
          // 这个publicPath会覆盖output中的
          publicPath: "./images/"
        }
      },
      {
        test: /\.css$/,
        use: [MiniExtractWebpackPlugin.loader, "css-loader"]
      }
    ]
  },
  devServer: {
    port: 8000,
    publicPath: "/dist/"
  },
  mode: "development",
  plugins: [
    new MiniExtractWebpackPlugin({
      filename: "[name].css",
      chunkFilename: "[id].css"
    })
  ]
};
```

将 css 打包到一个文件 默认是 main.css

##### 样式预处理

sass, less 预编译语言

- sass

使用 sass-loader 将 sass 转换为 css ，文件后缀名为.scss

搭配 cs-loader 和 style-loader 一起使用

除了安装 sass-loader 以外我们还需要安装 node-sass

node-sass 是真正用来编译 scss 的 而 sass-loader 只起到了连接作用

```
npm install sass-loader node-sass
```

```
      { test: /\.scss$/, use: ["style-loader", "css-loader", "sass-loader"] }
```

test.scss

```
body {
  h1 {
    background-color: bisque;
  }
}
```

打包后

```
body h1 {\\n  background-color: bisque; }
```

如果我们需要在调试工具中可以查看源码，我们需要配置 css-loader/sass-loader 的 sourceMap 选项

```
      {
        test: /\.scss$/,
        use: [
          "style-loader",
          {
            loader: "css-loader",
            options: {
              sourceMap: true
            }
          },
          {
            loader: "sass-loader",
            options: {
              sourceMap: true
            }
          }
        ]
      }
```

- less

```
npm i less-loader less
```

##### PostCSS

工作原理：接收样式源码交给编译插件处理，最后输出 CSS

postcss-loader

```
npm i postcss-loader
```

webpack.config.js

**注意顺序**

```
    {
        test: /\.css$/,
        use: ["style-loader", "css-loader", "postcss-loader"]
      }
```

需要一个配置文件

postcss.config.js

```
module.exports = {};
```

##### postcss 可以做的事情

1. 自动为样式添加游览器兼容前缀

autoprefixer

postcss.config.js

```
const autoprefixer = require("autoprefixer");
module.exports = {
  plugins: [
    autoprefixer({
      // 需要支持的特性
      grid: true,
      // 需要兼容的浏览器
      browsers: ["> 1%", "last 3 versions", "android 4.2", "ie 8"]
    })
  ]
};
```

打包前

```
div {
  display: grid;
}
```

打包后

```
div {\\n  display: -ms-grid;\\n  display: grid;\\n}
```

**postcss: 使用这个 loader 可以将 postCss 和 webpack 连接起来，必须要在根目录有 postcss.config.js，否则会报错。**

2. stylelint

css 语法检测

postcss.config.js

```
const stylelint = require("stylelint");
module.exports = {
  plugins: [
    stylelint({
      config: {
        rules: {
          "declaration-no-important": true
        }
      }
    })
  ]
};
```

3. cssnext

可以帮助我们在应用中使用最新的 css 语法

```
npm install postcss-cssnext
```

postcss.config.js

```
const postcssCssnext = require("postcss-cssnext");
module.exports = {
  plugins: [
    postcssCssnext({
      // 指定所支持的浏览器
      browers: [">1%", "last 2 versions"]
    })
  ]
};
```

style.css

```
:root {
  --highlightColor: lightblue;
}

body {
  background-color: var(--highlightColor);
}
```

打包后

```
body {\\n  background-color: lightblue;\\n}
```

##### css modules

将 css 进行模块化

特点

1. 每一个 css 模块都具有单独的作用域，不会和外界发生命名冲突

2. 对 css 进行依赖管理，可以通过相对路径引入 css

3. 可以通过 composes 轻松复用其他 css 模块

只需要对 css-loader 进行配置即可

```
      {
        test: /\.css$/,
        use: [
          "style-loader",
          {
            loader: "css-loader",
            options: {
              modules: true,
              localIdentName: "[name]_[local]_[hash:base64:5]"
            }
          },
          "postcss-loader"
        ]
      }
```

localIdentName 用于指明 css 类名会被如何编译

假设源码：

style.css

```
.title{
    font-size: 40px;
}
```

编译后

```
.style_title_1CFy6
```

- [name]指模块名，也就是文件名

- [local]指原本的 css 选择器的名称

- [hash:base64:5]指 5 位的 hash 值

使用方式

```
import styleCss from './style.css';
document.write('<h1 class=`${styleCss.title}`>Test<h1>')
```

> 在使用了 css modules 之后 import 的 css 文件返回的是一个 css 对象，所以需要把这个对象的属性作为类名
